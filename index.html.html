BATCH LIMIT

CRUNCH processes 50–60 rows per batch, always completing the current address group before stopping. Click NEXT BATCH to continue. Context is preserved between batches.

✕
CANCEL

Appears only while processing. Stops the current run immediately using an abort signal — no data is lost from previous batches.

🔄
RESET

Clears all batches, inputs, and output. Use when starting a completely new operator or dataset from scratch.

📋
COPY TABLE

Copies only the pipe-delimited rows — ideal for pasting directly into Smart List. Each table section also has its own inline Copy Table button.

📄
COPY ALL

Copies everything — flags, ambiguous notes, and all batch tables. Good for keeping a full processing record.

⬇️
DOWNLOAD CSV

Exports all pipe table rows as a .csv file, openable in Excel or Google Sheets for further editing.

⊞

Output columns

Columns are only included when the source data contains relevant information. Empty columns are suppressed entirely.

Address

Matched against reference CSV (85%+ match). Flagged as Confirmed or Best Guess.

Managed or Serviced

Only included when source has an explicit service type column. Never defaulted.

Floor/Unit/Office Number

Exact room or suite ID from source (e.g. 1.01, Unit 4, Ground Floor).

Office Type

Default: Private Office. Populated from source where available.

Internal or External

Only from explicit window/light/view/aspect references. Defaults to Office Space.

Capacity

Uses layout figure. Max Capacity column is always ignored. Ranges use max value.

Meeting Rooms

Integer only. Column omitted if no meeting room data in source.

Size

UK → Sq. Ft.  |  AU → Sqm. Column omitted if no size data present.

List Price Month to Month

Total monthly cost. Populated only when NO minimum term stated in source.

List Price 12 Month Term

Only populated when a minimum term is explicitly stated. Never defaulted.

Available

Available / Unavailable / Not Available. Missing dates → Available + flagged.

Details

Offer terms, min term notes. No per-desk workings or availability dates.

★

Tips & rules to know

📅
AVAILABILITY DATES

Now / current month / past months / within 6 months = Available. Beyond 6 months = Unavailable. AGREEMENT OUT / Under Offer / Let = Not Available.

👥
DESK RANGES

Ranges like "8 to 10", "8/10", "20/24" always use the maximum value in the Capacity column. Max Capacity columns are ignored entirely.

🪟
INTERNAL VS EXTERNAL

Only from explicit: window, natural light, external views, corner aspect, city/park/ocean/street view. Terrace, lounge, garden = Office Space.

💰
PRICING COLUMNS

If no minimum term stated anywhere → all prices go to Col I (M2M). Only moves to Col J when a term is explicitly stated. Per-desk rates × capacity only.

🏠
ADDRESS LOOKUP

1) Reference CSV (85%+ match) → 2) Rubberdesk site for address only → 3) Operator website for address only → 4) Best Guess flag. Address is never left partial.

🚫
EXCLUDED PRODUCTS

Day passes, virtual offices, and coworking memberships are excluded from the pipe table and flagged with their row reference in the flags table.

📦
DATA SOURCE ONLY

CRUNCH only extracts rows from the content you provide. It never fetches external listing pages or aggregators to supplement your data.

🔁
NEXT BATCH

For large datasets, CRUNCH processes 50–60 rows and stops at a clean address boundary. Hit NEXT BATCH to continue — all prior context is retained.
